# Cipi Agent for Laravel

Laravel package that integrates your application with [Cipi](https://github.com/andreapollastri/cipi) server control panel.

## What it does

- **Webhook endpoint** — receives push events from GitHub, GitLab, Bitbucket, Gitea (or any Git provider) and queues a deploy by writing a flag file. A cron running as the app user picks it up and runs Deployer. Zero sudo, zero permission issues.
- **Health check** — exposes a `/cipi/health` endpoint that reports app, database, Redis, cache, and queue status
- **Artisan commands** — `cipi:status` and `cipi:deploy-key` for quick diagnostics

## How deploy works

```
GitHub push → POST /cipi/webhook → agent verifies HMAC signature
                                  → agent writes /home/myapp/.deploy-trigger
                                  → returns 200 immediately

Cron (every minute, runs as myapp) → sees .deploy-trigger
                                    → deletes it
                                    → runs: dep deploy -f ~/.deployer/deploy.php
                                    → logs result to ~/logs/deploy.log
```

No sudo. No privilege escalation. PHP-FPM and the cron both run as the app user, so the flag file is readable/writable by both. Deployer also runs as the app user.

## Installation

```bash
composer require andreapollastri/cipi-agent
```

That's it. The service provider is auto-discovered.

If you want to customize the config:

```bash
php artisan vendor:publish --tag=cipi-config
```

## Configuration

When Cipi creates your app, it automatically adds these variables to your `.env`:

```env
CIPI_WEBHOOK_TOKEN=your-auto-generated-token
CIPI_APP_USER=your-app-username
CIPI_PHP_VERSION=8.4
```

You don't need to set them manually — Cipi handles it.

## Webhook Setup

### GitHub

1. Go to your repository → Settings → Webhooks → Add webhook
2. **Payload URL:** `https://yourdomain.com/cipi/webhook`
3. **Content type:** `application/json`
4. **Secret:** your `CIPI_WEBHOOK_TOKEN` value
5. **Events:** Just the push event

### GitLab

1. Go to your repository → Settings → Webhooks
2. **URL:** `https://yourdomain.com/cipi/webhook`
3. **Secret token:** your `CIPI_WEBHOOK_TOKEN` value
4. **Trigger:** Push events

### Bitbucket

1. Go to your repository → Settings → Webhooks → Add webhook
2. **URL:** `https://yourdomain.com/cipi/webhook`
3. **Triggers:** Repository push

### Gitea / Custom

The middleware supports HMAC signature verification (SHA256). Set the secret to your `CIPI_WEBHOOK_TOKEN`.

## Health Check

```bash
# With bearer token
curl -H "Authorization: Bearer YOUR_TOKEN" https://yourdomain.com/cipi/health

# With query string
curl https://yourdomain.com/cipi/health?token=YOUR_TOKEN
```

Response:

```json
{
  "status": "healthy",
  "app_user": "myapp",
  "php": "8.4",
  "laravel": "12.0.0",
  "environment": "production",
  "checks": {
    "app": { "ok": true },
    "database": { "ok": true, "database": "myapp" },
    "redis": { "ok": true },
    "cache": { "ok": true },
    "queue": { "ok": true, "connection": "redis", "pending_jobs": 0 }
  },
  "timestamp": "2025-01-15T10:30:00+00:00"
}
```

## Artisan Commands

```bash
# Show Cipi agent configuration and connectivity
php artisan cipi:status

# Show the SSH deploy key to add to your Git provider
php artisan cipi:deploy-key
```

## Branch Filtering

To only deploy when pushing to a specific branch, add to `.env`:

```env
CIPI_DEPLOY_BRANCH=main
```

Pushes to other branches will return a `skipped` response.

## Security

- Webhook requests are verified via HMAC signatures (GitHub, Gitea) or token comparison (GitLab, query string, Bearer)
- The webhook token is generated by Cipi during app creation and stored only in `.env`
- Health check requires authentication via the same token
- Deploy runs as the app's Linux user (not root)
